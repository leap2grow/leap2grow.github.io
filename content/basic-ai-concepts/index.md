  // Get contrast color;
  function getColorContrast(hexColor) {
    const rgb = hexToRgb(hexColor);
    let yiq = ((rgb[0] * 299) + (rgb[1] * 587) + (rgb[2] * 114)) / 1000;
    const colorContrast = (yiq >= 128) ? 'hsl(0 0% 0%)' : 'hsl(0 0% 100%)';

  };

  function hexToHsl(hexColor) {
    const rgb = hexToRgb(hexColor);

    // Make r, g, and b fractions of 1
    const r = rgb[0]/255;
    const g = rgb[1]/255;
    const b = rgb[2]/255;

    const max = Math.max(r, g, b), min = Math.min(r, g, b);
    let h, s, l = (max + min)  /  2;  
    if ( max == min ) { h = s = 0; } else {
      let d = max - min;
      s = l > 0.5 ? d / (2 - max - min) : d / (max + min);
      switch(max){
        case r: h = (g - b) / d + (g < b ? 6 : 0); break;
        case g: h = (b - r) / d + 2; break;
        case b: h = (r - g) / d + 4; break;
      }
      h /= 6;
    }

    return [Math.round(h * 360),Math.round(s * 100),Math.round(l * 100)]
  }

      <p><strong>It's no exaggeration</strong> to say that artificial intelligence is having sweeping impacts on our world. From the way consumers interact with technology, to the increasingly AI-driven basis in which most businesses are operating today, artificial intelligence is everywhere. </p><p>It's <em>also </em>only going to become more pervasive in the coming years, as large companies like Amazon and Google continue their <a href="https://link.springer.com/article/10.1007/s00146-022-01423-0?ref=nicksaraev.com">veritable space race</a> to dominate the AI market.</p><p>Because of its massive social and cultural relevance, it's worth learning a little bit about it. That's what we're going to do in this article - tread through a few basic AI concepts to make your next discussion (or business project) more likely to succeed.</p><h2 id="artificial-intelligence-what-is-it">Artificial intelligence: what is it?</h2><p>To begin our discussion on basic AI concepts, let's first define artificial intelligence.</p><div class="kg-card kg-callout-card kg-callout-card-accent"><div class="kg-callout-emoji">ðŸ’¡</div><div class="kg-callout-text"><strong>Artificial intelligence</strong>: the theory and development of computer systems able to perform tasks that normally require human intelligence, such as visual perception, speech recognition, decision-making, and translation between languages. (Oxford Dictionary)</div></div><p>Put simply, AI over the last sixty years has meant <strong>making our computers think more like humans</strong>. People are dynamic, flexible, and creative; computers, on the other hand, are incapable of doing more than following a set of very specific instructions. We want our systems to be more of the former and less of the latter.</p><p>With modern statistical techniques, this is possible. Today's <a href="https://en.wikipedia.org/wiki/Neural_network?ref=nicksaraev.com">neural network</a> based AI is inspired by, and in some respects already replicates, certain functions of the human brain. <a href="https://leap2grow.github.io/stylegan2-gpt3-duplex-replika-theres-no-going-back/">Though still limited in their capacity</a>, such nascent AI is definitive proof that we're moving towards flexible, autonomous systems.</p><figure class="kg-card kg-image-card"><img src="https://leap2grow.github.io/content/images/2022/03/jetsons.jpg" class="kg-image" alt loading="lazy" width="1400" height="1050" srcset="https://leap2grow.github.io/content/images/size/w600/2022/03/jetsons.jpg 600w, https://leap2grow.github.io/content/images/size/w1000/2022/03/jetsons.jpg 1000w, https://leap2grow.github.io/content/images/2022/03/jetsons.jpg 1400w" sizes="(min-width: 720px) 720px"></figure><h2 id="types-of-artificial-intelligence">Types of artificial intelligence</h2><p>The age-old adage of a walking, talking, feeling robot Ã  la 'The Jetsons' is, unfortunately, still a ways away. Instead, artificial intelligence engineers, until now, have spent the bulk of their time focusing on building AI models that can perform <em>very </em>well within a hyper-defined, small field or niche. </p><p>Such artificial intelligences are termed <strong>narrow AI</strong>, because they are good at doing only a few specific things. Understanding basic AI concepts like narrow AI is crucial if you want to be able to make sense out of the next decade, so we're going to talk a little bit about it.</p><h3 id="narrow-artificial-intelligence">Narrow artificial intelligence</h3><p>The artificial intelligences of today do certain things <em>very</em> well and other things not at all; i.e, their utility is <strong>narrow</strong>. But don't be fooled into thinking that <strong>narrow</strong> means <strong>not impressive </strong>- today's narrow AI is responsible for the vast majority of human dealings on the Internet, by syndicating, curating, and monitoring content on social media platforms, and some could say it's already driving our culture.</p><p>Other narrow artificial intelligences already in use include computer vision models, which already outperform human beings at a variety of recognition and identification tasks using digital images or videos. We've also created <a href="https://openai.com/five/?ref=nicksaraev.com">narrow game models</a>, narrow language processing models, <a href="https://github.com/NVlabs/stylegan?ref=nicksaraev.com">narrow generative models</a>, and more. </p><p>Though these narrow artificial intelligences can certainly best a human in the activity they were specifically designed to do, they often struggle when faced with tasks that are even slightly different to the ones they were originally trained for. So despite their incredible performance on object recognition tasks, for example, a computer vision model can't (yet) write, talk, speak, or develop software, because this was not explicitly trained for.</p><h3 id="general-artificial-intelligence">General artificial intelligence</h3><p>Narrow artificial intelligence stands in contrast with its significantly more formidable cousin: <strong>general artificial intelligence</strong> (AGI). AGI is the stuff of science fiction, and is characterized by intelligence that is not narrowly focused on a specific task, but can instead handle a wide range of cognitive functions similar to (or better than) humans. This is one of the most important basic AI concepts to understand because of its catastrophic potential impact. </p><p>In theory, general AI would be capable of outperforming humans in just about every intellectual endeavor - including activities that are currently seen as the exclusive domain of human beings, such as creativity, strategic thinking, and social skills.</p><p>This is for several reasons. The biggest may be that human neural impulses are limited to travelling at <a href="https://nba.uth.tmc.edu/neuroscience/m/s1/chapter03.html?ref=nicksaraev.com#:~:text=A%20great%20variability%20is%20found,(0.22%20miles%20per%20hour).">~100 meters per second</a>, whereas the speed of conduction in an artificial mind is closer to <a href="https://www.google.com/search?q=speed+of+light&rlz=1C1GEWG_enCA957CA957&sxsrf=APq-WBuGFxUuJh2ecpeynT-X_I8eHAcCjg%3A1647306284512&ei=LOYvYvvmHoT49AO1-orYBQ&ved=0ahUKEwj7rLKP9sb2AhUEPH0KHTW9AlsQ4dUDCA4&uact=5&oq=speed+of+light&gs_lcp=Cgdnd3Mtd2l6EAMyBAgjECcyBQgAEJECMgUIABCRAjIECAAQQzIECAAQQzIFCAAQgAQyBQgAEIAEMgUIABCABDIFCAAQgAQyBQgAEIAEOgcIABBHELADOgcIIxCxAhAnOgQIABAKSgQIQRgASgQIRhgAUKIBWKgGYOQGaAJwAXgAgAFciAG1ApIBATSYAQCgAQHIAQjAAQE&sclient=gws-wiz&ref=nicksaraev.com">300,000,000 meters per second</a> (the speed of light). </p><p>That's a <strong>3,000,000x</strong> speedup alone; meaning that if an AGI was theoretically <em>just</em> as smart as a human (and its mind was built in a similar way to ours), it would <em>still</em> be capable of accomplishing tasks close to three million times faster. A thousand or two human-level artificial intelligences could replace the economic value of our entire planet.</p><p>Another is that biological evolution may have left a fair amount on the table, optimization-wise. <a href="https://www.lesswrong.com/posts/xwBuoE9p8GE7RAuhd/brain-efficiency-much-more-than-you-wanted-to-know?ref=nicksaraev.com">There are good arguments to the contrary</a>, but many believe that <a href="https://en.wikipedia.org/wiki/Gradient_descent?ref=nicksaraev.com">gradient descent</a>, the main optimization algorithm employed by most statistical machine learning models today, probably optimizes neuron weights more efficiently than the way human brains do.</p><p>Which naturally leads to...</p><h3 id="the-singularity">The Singularity</h3><p><a href="https://www.nickbostrom.com/superintelligence.html?ref=nicksaraev.com">Many prominent scientists</a> believe such an AGI will be developed in the next few decades, and that it will have a devastating (or liberating) effect on humankind, going as far as to dub it the <a href="https://en.wikipedia.org/wiki/Technological_singularity?ref=nicksaraev.com"><strong>Singularity</strong></a>.</p><p>Perhaps the most apocryphal of basic AI concepts, the Singularity is the notion that once artificial intelligence surpasses human intelligence, it will rapidly evolve and improve at an accelerating rate, eventually leading to a point where humans will no longer be able to understand or control it.</p><p>Some proponents of the singularity believe this event will mark the end of humanity as we know it, while others see it as an opportunity for us to achieve a state of unimaginable prosperity and technological achievement.</p><p>If you assume a sufficiently rapid rate of technological improvement, it's not hard to see how a singularity could come about. In fact, some people believe it's already happening - with exponential increases in computing power and data storage, artificial intelligence is rapidly getting better at learning and problem solving. </p><p>AI is already <a href="https://openai.com/blog/openai-codex/?ref=nicksaraev.com">writing code</a>, <a href="https://www.bostondynamics.com/?ref=nicksaraev.com">learning to run</a>, and more - the gap between human and artificial intelligence is rapidly shrinking, and will continue to do so over the coming decades.</p><h3 id="where-ai-progress-is-today-foundation-models">Where AI progress is today: foundation models</h3><p>As we grow further into the digital age, models are growing less and less narrow. This has necessitated the invention of a new basic AI concept: the <strong><a href="https://arxiv.org/abs/2108.07258?ref=nicksaraev.com">foundation model</a></strong>.</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://leap2grow.github.io/content/images/2022/03/image-1.png" class="kg-image" alt loading="lazy" width="1157" height="233" srcset="https://leap2grow.github.io/content/images/size/w600/2022/03/image-1.png 600w, https://leap2grow.github.io/content/images/size/w1000/2022/03/image-1.png 1000w, https://leap2grow.github.io/content/images/2022/03/image-1.png 1157w" sizes="(min-width: 720px) 720px"><figcaption>From "<a href="https://arxiv.org/pdf/2108.07258.pdf?ref=nicksaraev.com">On the Opportunities and Risks of Foundation Models</a>"</figcaption></figure><p>Foundation models are chiefly distinguished by their adaptability; whereas narrow AI is restricted to one or a small set of very specific tasks, foundation models are beginning to achieve multimodality: the ability to understand a diverse medium of data (like text, images, <em>and </em>video simultaneously) rather than just the one it was explicitly encoded for.</p><p>The most recent (and perhaps impactful) development of the sort is in <strong>natural language processing</strong>, a subfield of AI that deals with teaching computers how to understand human language. </p><p><a href="https://openai.com/blog/openai-api/?ref=nicksaraev.com">GPT-3</a>, a model released by <a href="https://openai.com/?ref=nicksaraev.com">OpenAI</a> in late 2019, was initially trained for a single task - understanding and generating text. However, after being released to the Internet, users quickly found that it far exceeded its initial training set; <a href="https://www.nytimes.com/2020/11/24/science/artificial-intelligence-ai-gpt3.html?ref=nicksaraev.com">GPT-3 can do math, understand logic, and, with some tweaks, even create images</a> with a high degree of fluency. Not to mention its original task - text generation - is oftentimes indistinguishable from human writing.</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://leap2grow.github.io/content/images/2022/03/openai-cover.png" class="kg-image" alt loading="lazy" width="2000" height="1125" srcset="https://leap2grow.github.io/content/images/size/w600/2022/03/openai-cover.png 600w, https://leap2grow.github.io/content/images/size/w1000/2022/03/openai-cover.png 1000w, https://leap2grow.github.io/content/images/size/w1600/2022/03/openai-cover.png 1600w, https://leap2grow.github.io/content/images/2022/03/openai-cover.png 2400w" sizes="(min-width: 720px) 720px"><figcaption><a href="https://openai.com/?ref=nicksaraev.com">OpenAI</a>, the developers of GPT-3.</figcaption></figure><p>To illustrate this, I wrote this article, in part, using GPT-3. It excels at explaining straightforward topics in an easy-to-understand way, and though there are (at times) factual inaccuracies, it significantly improves the speed at which one can create. </p><p>You can liken the usage of such a tool to the <a href="https://en.wikipedia.org/wiki/Power_loom?ref=nicksaraev.com">power loom</a> during the industrial revolution - the efficiency of a single weaver (or writer) is instantly multiplied by a dozen or more times. Instead of having to do the grunt work, you're involved more in <em>steering the result where you want it to go</em>. Over the next ten to twenty years, it's suspected that many large industries (driving, advertising, and marketing, to name a few) will follow suit, and this will have significant impacts on the labor market.</p><h2 id="the-mathematics-of-artificial-intelligence">The mathematics of artificial intelligence</h2><p>Far from a basic AI concept, the mathematics that underpin artificial intelligence are diverse and complex. Though beyond the scope of this article, it's important to understand that AI math really consists of a wide range of concepts and tools, from linear algebra, to probability and statistics, to game theory, and more. We'll cover the first two - linear algebra and statistics - in brief below.</p><h3 id="linear-algebra">Linear algebra</h3><p>Linear algebra is important for AI because it provides a way to model relationships between data points. In particular, linear algebra allows agents to represent data in terms of <a href="https://en.wikipedia.org/wiki/Matrix_(mathematics)?ref=nicksaraev.com">vectors and matrices</a>, and extend this notion to higher dimensions. </p><p>Matrices are at the crux of how agents identify patterns and relationships in data. Almost all machine learning can reasonably be described as matrix operations - <a href="https://en.wikipedia.org/wiki/Transformer_(machine_learning_model)?ref=nicksaraev.com">the Transformer architecture underlying GPT-3</a>, for example, is nothing but large matrices multiplied together several times to produce another matrix that describes a sentence.</p><p>That said, linear algebra can do very little without statistics &amp; probability. Because the way your algorithm is <em>trained, </em>i.e, statistically optimized, is at the root of how models learn.</p><h3 id="probability-and-statistics">Probability and statistics</h3><p>Probability and statistics are essential artificial intelligence basis to understand. Any AI system that needs to make decisions based on data, and <em>then update based on those decisions, heavily </em>relies on both concepts.</p><p>Probability provides a way to quantify uncertainty, while statistics allows agents to analyze data and draw conclusions from it - even data that may not traditionally seem 'analyzable', like words, images, or sounds.</p><p>In particular, two concepts in statistics are important to understand: optimization and Bayes' theorem. Optimization refers to the notion of iteratively improving an estimate through the use of approximation methods like <a href="https://towardsdatascience.com/gradient-descent-algorithm-a-deep-dive-cf04e8115f21?ref=nicksaraev.com#:~:text=Gradient%20descent%20(GD)%20is%20an,e.g.%20in%20a%20linear%20regression).">gradient descent</a>.</p><p><a href="https://en.wikipedia.org/wiki/Bayes%27_theorem?ref=nicksaraev.com">Bayes' theorem</a> is a way of applying probability to conditional statements. This is important for AI because it allows agents to update their beliefs about the world as they receive new information. </p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://leap2grow.github.io/content/images/2022/03/s25RsOr---Imgur.gif" class="kg-image" alt loading="lazy" width="513" height="512"><figcaption>A series of animations from Alec Radford showing how optimization updates towards the end result.</figcaption></figure><p>Optimization and Bayes' theorem both work together to provide systems with a way of <em>moving in the right direction</em>, so to speak. You get 'closer' to the desired result by moving in a direction, calculating your distance to the end result, and then moving again in a way that makes that distance smaller.</p><h3 id="basic-ai-concepts-where-do-i-go-from-here">Basic AI concepts: where do I go from here?</h3><p>At this point, you should have a reasonable understanding of some of the core basic AI concepts, including narrow AI, AGI, the Singularity, and the mathematics required to do AI. If you're interested in learning more, I'd recommend checking out some of the following resources:</p><ul><li><strong><a href="http://themlbook.com/?ref=nicksaraev.com">The 100 Page Machine Learning Book by Andriy Burkov</a>: </strong>This online resource provides concise explanations of a wide range of artificial intelligence basics, and is pivotal if you wish to develop an understanding of the statistical side of AI; i.e machine learning.</li><li><strong><a href="https://www.coursera.org/learn/ai-for-everyone?ref=nicksaraev.com">AI for Everyone</a></strong>: Though non-technical in nature, free online course from Andrew Ng provides a more comprehensive introduction to many of the basic AI concepts we've discussed here.</li><li><strong><a href="https://leap2grow.github.io/">This blog</a></strong>: Over the coming weeks and months, I'm going to be creating in-depth resources on how to get up and running with AI (moreso than the basic AI concepts covered here). We'll learn how to create AI artwork, how AI text generation works, why AI deepfakes aren't that big of a deal, and more, and I'm going to keep it palatable to a wide audience while also offering the opportunity for people to go deeper if interested.</li></ul>

    function renderSubmenus() {
      const navItems = document.querySelectorAll('[data-nav] li')
      const subItems = document.querySelectorAll('.is-subitem')
      const mobileClass = "hidden pl-4 flex flex-col gap-0.5 py-1 px-1.5 basis-full"
      const desktopClass = "z-[50] min-w-[calc(100%_+_24px)] absolute flex flex-col gap-1 shadow-2xl border border-brd rounded-theme-xxs bg-bgr py-2 px-1.5 -ml-2 mt-11 opacity-0 invisible translate-y-1 transition-all"

      // Remove '-' signs
      subItems.forEach(item => {
        const itemName = item.querySelector('a span')
        itemName.innerText = itemName.innerText.slice(1)
      });

      // Add subitems in place
      let subMenu, hasItems
      navItems.forEach((item, index) => {
        if (item.classList.contains('is-subitem') && !navItems[index - 1].classList.contains('is-subitem')) {
          navItems[index - 1].classList.add('is-mainitem'); 
          navItems[index - 1].setAttribute('x-on:click.outside', 'closeSubmenus(event)')
        }
        subMenu = item.classList.contains('is-subitem') ? subMenu : document.createElement('ul');
        if (item.classList.contains('is-subitem')) { 
          subMenu.appendChild(item)
          subMenu.setAttribute('data-submenu', '')
          subMenu.getAttribute('id') === null ? subMenu.setAttribute('id', navItems[index - 1].getAttribute('data-slug')) : ''
          const menuType = subMenu.closest('nav').getAttribute('data-nav')
          subMenu.classList = menuType === 'desktop' ? desktopClass : mobileClass
        } else { 
          item.appendChild(subMenu)
        }
      });

      const dropdownMenus = document.querySelectorAll('[data-nav] .is-mainitem')
      const toggle = document.querySelector('[data-toggle-template]');

      dropdownMenus.forEach(menu => { 
        const toggleBtn = toggle.content.firstElementChild.cloneNode(true);
        toggleBtn.setAttribute('aria-controls', menu.getAttribute('data-slug'))
        menu.insertBefore(toggleBtn, menu.children[1]);
      });
    }
